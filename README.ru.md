# Relay MQ

[English](README.md) | Русский

`Relay MQ` — это брокер событий для асинхронного взаимодействия между сервисами.
Проект вдохновлен системами вроде Kafka и Redis Streams и фокусируется на низкой задержке, надежности и горизонтальном масштабировании.

## Описание

Relay MQ помогает строить event-driven архитектуру, где продюсеры публикуют события, а консюмеры обрабатывают их независимо друг от друга.

Основные сценарии:
- обмен событиями между сервисами
- асинхронная обработка команд и задач
- fan-out событий на несколько потребителей
- надежная доставка сообщений в распределенных системах

## Цели Проекта

- предсказуемая доставка сообщений под нагрузкой
- удобное горизонтальное масштабирование за счет добавления узлов
- отказоустойчивость и восстановление после перезапуска
- понятный и компактный протокол для клиентов
- практичность в development и production

## Ключевые Возможности

- модель publish/subscribe
- поддержка consumer groups
- настраиваемая семантика доставки
- кластеризация и репликация данных
- персистентное хранение для восстановления
- административные и наблюдаемые endpoints

## Статус Разработки

Проект находится в активной разработке.
Базовая функциональность брокера уже реализована, но API и поведение отдельных частей могут меняться.

## Быстрый Старт

```bash
python3 main.py
```

По умолчанию Relay MQ запускает:
- брокер-сервер
- endpoint потока событий
